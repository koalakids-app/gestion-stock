🐨 Migration photos → Supabase Storage

Outil à usage unique : transfère les photos stockées en base64 dans la table articles vers le bucket assets, et remplace la valeur en base par l'URL publique.

1 Connexion à Supabase

Tu la trouves dans ton fichier index.html de l'app, là où est défini SUPABASE_URL (ou dans createClient(...)).
Même clé que celle utilisée dans ton app pour createClient(url, clé). Cette clé reste dans ton navigateur, elle n'est envoyée nulle part ailleurs.

2 Analyser la table

Cette étape ne modifie rien. Elle parcourt la table articles et repère, pour chaque ligne, les colonnes qui contiennent une photo en base64 (longue chaîne data:image/... ou base64 brut).

3 Lancer la migration

Pour chaque photo trouvée : upload dans le bucket assets (dossier articles-photos/), puis la colonne en base est remplacée par l'URL publique. Réversible si tu as une sauvegarde

Conseil : avant de lancer, fais un export CSV de la table articles depuis le Table Editor de Supabase (bouton "Export" en haut), juste par sécurité.
